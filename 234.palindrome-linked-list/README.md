一个比较慢但是容易想到的方法是把原链表反转存入一个新的链表，然后判断两个链表是否相同。反转的方法可以参考[206题](../206.reverse-linked-list/206.%E5%8F%8D%E8%BD%AC%E9%93%BE%E8%A1%A8.py)，稍加一些修改

第二个方法可以使用链表的后序遍历方法，核心逻辑就是把链表节点放入一个栈，然后再拿出来，这时候元素顺序就是反的，就可以拿来进行判断

上面两个方法的时间和空间复杂度都是O(N)

第三个方法的空间复杂度可以优化为O(1)，思路是先通过[快慢指针法](../876.middle-of-the-linked-list/876.%E9%93%BE%E8%A1%A8%E7%9A%84%E4%B8%AD%E9%97%B4%E7%BB%93%E7%82%B9.py)找到链表的中点，假设链表是回文链表，如果链表个数为奇数，那么从中点那里分段，前半部分链表和后半部分链表应该是相同但反序的，这时我们可以将后半部分链表进行一次反序，然后比较；同理如果链表个数是偶数，也是比较前后两半部分链表
